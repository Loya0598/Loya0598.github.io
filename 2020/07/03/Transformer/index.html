Simple Intro

  • A model that uses attention to boost the speed
  • Outperforms the Google Neural Machine Translation model
  • Parallelization

A High-Level Look

  • Sentence1 $\rightarrow$ Model $\rightarrow$ Sentence2

  • Sentence1 $\rightarrow$ Encoder $\rightarrow$ Decoder $\rightarrow$ Sentence2

  • Sentence1 $\rightarrow$ Stack(Encoder, 6) $\rightarrow$ Stack(Decoder, 6) $\rightarrow$ Sentence2

    • Encoders don’t share weights

  • Sentence1 $\rightarrow$ Stack(Self-Attention + FFNN, 6) $\rightarrow$ Stack(Self-Attention + Encoder-Decoder Attention + FFNN, 6) $\rightarrow$ Sentence2

    Question1: The exact same feed-forward network is independently applied to each position.

    Answer1: For each position, FFNN is the same, so it can be done in parallel.

Bringing The Tensors Into The Picture

  • Encoders

    • Encoder Structure

      • Input: [Batch_Size, Seq_Len, Hidden_Dim1]
      • Layers: Self-Attention Layer + feed forward Layer
      • Output: [Batch_Size, Seq_Len, Hidden_Dim2]

    • Self-Attention:

      • Concept: Using the representation of other positions to encode the word that is processing now.

      • Details:

        • Input Vector $x$ / Query Vector $q$ / Key Vector $k$ / Value Vector $v$

        • First Step:

          • Query Vector $q = x * W^Q$

          • Key Vector $k = x * W^K$

          • Value Vector $v = x * W^V$

          • $W^Q, W^K, W^V$ are the parameters trained during the training process

            Dimension of new vectors: 64

            Dimension of input vectors: 512

            They don’t HAVE to be smaller, this is an architecture choice to make the computation of multiheaded attention (mostly) constant

        • Second Step:

          • Score: When processing the word in position i, the score at position j is Query Vector $q_i$ Key Vector $k_j$ (dot product)

        • Third Step:

          • divide the scores by $d$, $d = \sqrt{d_k}$

        • Forth Step:

          • Score $\rightarrow$Softmax Layer

        • Fifth Step:

          • Softmax Score * Value vector

        • Sixth Step:

          • Sum up the weighted value vectors

      • Matrix Form:

        • Input Data: $X$
        • Query Vector: $Q = XW^Q$
        • Key Vector: $K = XW^K$
        • Value Vector: $V = XW^V$
        • Output: $Z = softmax(\frac{QK^T}{\sqrt{d_k}})V$

      • “Many Heads”

        • Improvements:
          • Expands the model’s ability to focus on different positions
          • Gives the attention layer multiple “representation subspaces”
            • Using multiple sets of Query/Key/Value weight matrices
        • Implementation:
          • How to condense multiple outputs:
            • Concatenate and multiply them by an additional weight matrix $W^O$
            • Formula: $O = Concat(Z) * W^O$

      Answer2: Interpret the weight of the head one by one?

The Residuals

  • Each sub-layer(Self-Attention, FFNN) has a residual connection around it. And it is followed by a layer-normalization step
  • Formula: $Output = LayerNorm(X + Z)$

The Decoder Side

  • Self Attention Layer

    • Aim: give attention on the earlier positions in the output sequence
    • Implementation: mask future position before the softmax step

    Question4: How to implement it in parallel, it looks like it can only process one word at each forward step.

    Answer4: Training - parallel

  • Encoder-Decoder Attention Layer

    • Using the output of the top encoder to calculate the attention weight of input
    • Formula: $K = XK_{enc_dec}$, $V = XV_{enc_dec}$

    Question5: What’s the query vector?

    Answer5: From the layer below it

  • Output Layer: Linear + Softmax
